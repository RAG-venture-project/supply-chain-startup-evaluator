# Supply Chain Startup Investment Evaluation Agent

본 프로젝트는 Supply Chain 스타트업에 대한 투자 가능성을 자동으로 평가하는 멀티 에이전트 시스템입니다.

## Overview

- **Objective** : Supply Chain 스타트업의 기술력, 시장성, 경쟁 우위, 팀 역량을 기준으로 투자 적합성 분석
- **Method** : Multi-Agent + Agentic RAG (LangGraph 기반 워크플로우)
- **Tools** : LangGraph, FAISS, BGE-M3, GPT-4o-mini

## Features

- PDF 자료 기반 정보 추출 (기술 리포트, 시장 분석, 경쟁사 비교, 팀 보고서)
- 체크리스트 기반 정량 평가 (카테고리별 5문항 × 4개 영역)
- 가중 스코어링 투자 판단 (팀 35%, 시장 30%, 기술 20%, 경쟁 15%)
- 투자 추천 보고서 / 전체 보류 보고서 자동 생성

| --- | ----------- | --------- | --------------------------------------- |
| 1   | 기술 요약   | RAG Agent | 제품/기술력 체크리스트 평가 (20%)       |
| 2   | 시장성 평가 | RAG Agent | 시장 규모·성장성 체크리스트 평가 (30%)  |
| 3   | 경쟁사 비교 | RAG Agent | 경쟁 우위·차별점 체크리스트 평가 (15%)  |
| 4   | 팀 평가     | RAG Agent | 창업자·팀 역량 체크리스트 평가 (35%)    |
| 5   | 투자 판단   | LLM Agent | 4개 결과 종합 스코어링 + 추천/보류 판단 |

## 투자 판단 평가 기준

Bessemer's Investment Checklist와 Scorecard Method를 기반으로 설계하였다.  
각 에이전트가 5개 체크리스트 항목을 Yes/No로 판정하며, 항목당 20점 (카테고리 만점 100점)으로 환산한다.

### 1. 창업자/팀 (35%)

| #   | 질문                                                | 판단 기준                     |
| --- | --------------------------------------------------- | ----------------------------- |
| 1   | 창업자가 해당 도메인 경력 3년 이상인가?             | 물류/로봇/공급망 직접 경력    |
| 2   | 핵심 기술진(CTO 등)이 별도로 존재하는가?            | 대표와 기술 리더가 분리       |
| 3   | 팀원 중 글로벌 기업(아마존, 구글 등) 출신이 있는가? | 관련 대기업 경력자 1명 이상   |
| 4   | 현재 팀 규모가 30명 이상인가?                       | 실행력 확보 여부              |
| 5   | 주요 경영진 이탈 없이 안정적인가?                   | CEO 교체, 대량 해고 이력 없음 |

### 2. 시장성 (30%)

| #   | 질문                                           | 판단 기준           |
| --- | ---------------------------------------------- | ------------------- |
| 1   | TAM이 $10B 이상인가?                           | 목표 시장 규모      |
| 2   | 해당 시장의 CAGR이 15% 이상인가?               | 시장 리포트 기준    |
| 3   | 실제 매출 또는 유료 고객이 존재하는가?         | 매출 $0 초과        |
| 4   | 글로벌 시장(2개국 이상)에 진출했는가?          | 해외 법인·고객 보유 |
| 5   | 최근 1년 내 매출 또는 고객 수가 증가 추세인가? | YoY 성장 확인       |

### 3. 제품/기술력 (20%)

| #   | 질문                                                  | 판단 기준                   |
| --- | ----------------------------------------------------- | --------------------------- |
| 1   | 상용화된 제품(MVP 이상)이 존재하는가?                 | PoC가 아닌 실제 운영        |
| 2   | 특허를 1건 이상 보유하고 있는가?                      | 등록 또는 출원              |
| 3   | 기존 인프라 변경 없이 도입 가능한가?                  | 고객 전환 비용 낮음         |
| 4   | AI/자동화 등 핵심 기술을 자체 개발했는가?             | 외부 의존 아닌 자체 기술    |
| 5   | 고객사에서 측정 가능한 성과(KPI 개선)가 입증되었는가? | 생산성 향상, 비용 절감 수치 |

### 4. 경쟁 우위 (15%)

| #   | 질문                                                            | 판단 기준             |
| --- | --------------------------------------------------------------- | --------------------- |
| 1   | 동일 솔루션을 제공하는 직접 경쟁사 대비 명확한 차별점이 있는가? | 기술·가격·모델 차이   |
| 2   | 대형 고객사(Fortune 500 또는 동급)를 확보했는가?                | 레퍼런스 고객         |
| 3   | 네트워크 효과 또는 데이터 해자(moat)가 있는가?                  | 쓸수록 강해지는 구조  |
| 4   | 전환 비용(switching cost)이 높아 고객 이탈이 어려운가?          | 시스템 연동·종속성    |
| 5   | 최근 2년 내 전략적 파트너십(대기업·플랫폼)을 체결했는가?        | 공식 제휴·SPN 선정 등 |

### 스코어링

- 항목당 배점: Yes = 20점, No = 0점 (카테고리 만점 100점)
- 총점 = 팀(×0.35) + 시장(×0.30) + 기술(×0.20) + 경쟁(×0.15)
- 60점 이상 → **투자 추천**, 미만 → **보류**

│   │   ├── market_eval.py         # 시장성 평가 에이전트
│   │   ├── competitor.py          # 경쟁사 비교 에이전트
│   │   ├── team_eval.py           # 팀 평가 에이전트
│   │   ├── investment.py          # 투자 판단 에이전트
│   │   └── report.py              # 보고서 생성 에이전트
│   ├── graph/
│   │   └── workflow.py            # LangGraph 워크플로우
│   ├── schemas/
│   │   ├── state.py               # InvestmentState 정의
│   │   └── output.py              # 에이전트 출력 포맷
│   ├── vectorstore/
│   │   ├── embeddings.py          # BGE-M3 임베딩
│   │   ├── loader.py              # PDF 로더 + 청킹

## Setup

```bash
# 1. 가상환경 생성 및 활성화
python -m venv .venv
source .venv/bin/activate

# 2. 패키지 설치
pip install -r requirements.txt

# 3. 환경변수 설정
cp .env.example .env
# .env 파일에 OPENAI_API_KEY 입력

# 4. FAISS 인덱스 적재
python -m scripts.ingest

# 5. 실행
python main.py
```
